Server Notifications & Alerts

Never miss another server outage. Learn how to build a server notification system that will alert all administrators via SMS when a server outage occurs.

Start Tutorial

What does this thing do?

This Express.js sample application demonstrates how to send SMS alerts to a list of system administrators if something goes wrong on your server. In this tutorial, we'll show you the key bits of code to make this work. Head to the application's README.md to see how to run the application locally.

See how EMC uses Twilio SMS to send IT alerts to 68,000 employees.

Let's get started!

See Also:

  • Getting started with Express.js
  • twilio-node library

Configure Twilio client

In order to send messages we'll need to create a Twilio REST client, which requires reading our TWILIO_ACCOUNT_SID, TWILIO_NUMBER TWILIO_AUTH_TOKEN from the enviroment. We also make the variables available through the config module.

See Also:

  • twilio-node Quickstart
  • Twilio REST Web Service Interface

List server admins

Here we create a list of administrators who should be notified if a server error occurs. The only essential piece of data we need is a phoneNumber for each administrator.

Handle all the exceptions in the application

We will implement error handling and message delivery as a piece of Express.js middleware. We will make all the Twilio API calls from there.

See Also:

  • Error handling middleware

Trigger notifications for administrator list

In our Express middleware module, we read the admins from our JSON file and send alert messages to each of them with the sendSms method in twilioClient. We also call next with the error object, which will allow other configured error handling middleware to execute.

Send a text message

There are the three parameters needed to send an SMS using the Twilio REST API: From, To, and Body. US and Canadian phone numbers can also send an image with the message.
