Server Notifications

Never miss another server outage. Learn how to build a server notification system that will alert all administrators via SMS when a server outage occurs.

Start Tutorial

What does this thing do?

This Java Servlets sample application demonstrates how to send SMS alerts to a list of system administrators if something goes wrong on your server. In this tutorial, we'll show you the key bits of code to make this work.

See how EMC uses Twilio SMS to send IT alerts to 68,000 employees.

Let's get started!

  • What is a Servlet?

List server admins

Create a list of administrators who should be notified if a server error occurs. Each administrator must have a phoneNumber.

Configure Twilio client

In order to send a message we'll need to initialize the Twilio Java Helper Library, which requires reading our TWILIO_ACCOUNT_SID and TWILIO_AUTH_TOKEN defined as environment variables.

TwilioMessageCreator is a custom wrapper for MessageCreator.

  • Twilio Java SDK

Handle all errors

Using a Servlet Filter, we can capture any unhandled exceptions thrown by any other Servlets or Filters set up to handle a given URL. This would be our opportunity to send out an alert message to administrators.

Create a custom alert message

Here we create an alert message to send out via text message. You might also decide to include a picture with your alert message, maybe a screenshot of the application when the crash happened?

Reading the admins from the JSON file

Next we read the admins from our JSON file. We're relying on the Gson Java library to convert our JSON text file into Administrator objects, which we created in our application.

Sending a text message

There are the three parameters needed to send an SMS using the Twilio REST API: From, To, and Body. US and Canadian phone numbers can also send an image with the message.
