    How to set up your C# and ASP.NET MVC development environment

    Agustin Camino

    In this guide we’ll cover how to set up your ASP.NET development environment for an MVC project. We’ll also talk about a couple of helpful tools that we recommend for all C# applications that use Twilio: ngrok and the Twilio C# SDK.

    Let’s get started!

    Install the .NET Framework and an IDE

    Operating System Instructions
    Windows For Windows, we recommend to use Visual Studio Community Edition, which is a free IDE provided by Microsoft. It will install all the necessary component as well as the .NET Framework.
    Mac OS X The recommended way to work with .NET projects in this platform is to use Xamarin Studio which includes the Mono Framework (and a brief description about what is that thing.
    Linux/BSD We recommend MonoDevelop that you can install through your usual package manager.

    However for the rest of this guide, we will use Visual Studio on Windows.

    Create a simple ASP.NET MVC application

    First let’s create a simple ASP.NET MVC project. Visual Studio can create and fill everything for you in a few simple clicks.

    1. In the Menu click File > New > Project
    2. This will open a dialog, in the Templates for Visual C# choose ASP.NET Web Application (.NET Framework)
    3. At the bottom enter the name for the project such as WebApplicationTwilio and click Ok
    4. In the next dialog select MVC, uncheck the box Host in Azure and click Ok

    And that’s you should have a fully set up ASP.NET MVC project ready to go. You can click in the menu bar on Debug > Start Debugging (or press F5) it will automatically build, run the application and open your web browser to it. (Don’t forget to stop the debugging before the next step).

    Using Twilio C# SDK in an ASP.NET MVC Project

    Now we will add the Twilio C# SDK and show a simple example on how to use it with an MVC project.

    1. In the Solution Explorer right click on your Solution and select Manage NuGet Packages for Solution
    2. This will open a new tab showing you the installed packages of your Solution. Click on Browse
    3. Make sure that include prerelease is checked and Search for Twilio
    4. Select Twilio by Twilio, on the right hand pane check the box for your project choose the latest version and click Install

    The next step will be to modify the HomeController which handle what the application returns when you first connect to it. Right now it returns a basic HTML webpage, we will change that to returns some TwiML.

    Replace the content of HomeController.cs by the following

    using System.Web.Mvc;
using Twilio.TwiML;

namespace WebApplicationTwilio.Controllers
{
    public class HomeController : Controller
    {
        public ActionResult Index()
        {
            var response = new VoiceResponse();
            response.Say("hello world");
            return Content(response.ToString(), "text/xml");
        }
    }
}

    And that’s it, you can press F5 and you should see a page displaying

    <Response>
    <Say>hello world</Say>
</Response>

    Installing and using ngrok

    Now that you have a simple "Hello World!" in TwiML, your development environment is ready to go. But for most Twilio projects you’ll want to install one more helpful tool: ngrok.

    Most Twilio services use webhooks to communicate with your application. When Twilio receives an incoming phone call, for example, it reaches out to a URL in your application for instructions on how to handle the call.

    When you’re working on your application in your development environment, your app is only reachable by other programs on the same computer, so Twilio won’t be able to talk to it.

    ngrok is our favorite tool for solving this problem. Once started, it provides a unique URL on the ngrok.io domain which will forward incoming requests to your local development environment.

    To start, head over to the ngrok download page and grab the binary for your operating system: https://ngrok.com/download

    Once downloaded, right click on your project select properties in the contextual menu then in the Web section change the Project Url to http://localhost:3000 and Save. Make sure your ASP.NET MVC application is running and then start ngrok using this command: ./ngrok.exe http 3000. You should see output similar to this:

    Look at the "Forwarding" line to see your unique ngrok domain name (ours is "aaf29606.ngrok.io") and then point your browser at that domain name.

    If everything’s working correctly, you should see your “Hello World!” message displayed at your new ngrok URL.

    Anytime you’re working on your Twilio application and need a URL for a webhook you should use ngrok to get a publicly accessible URL like this one.

    Configuring a webhook on Twilio

    For the final step, heads up towards the phone number section in Twilio’s console.

    1. Set the "Configure with" to Voice Webhook/TwiML
    2. Fill in your ngrok URL
    3. Configure it as a HTTP GET
    4. At the bottom click on the Save button

    That’s it, you can take your phone and call your Twilio Number, you should hear your "Hello World!"
