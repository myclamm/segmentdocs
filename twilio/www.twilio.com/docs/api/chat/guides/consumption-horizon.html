    Message Consumption Horizon and Read Status

    The Programmable Chat Message Consumption Horizon feature allows a User's Message read status to be synchronised across devices and endpoints. This is done by referencing the User's consumption or read to point within a Channel's Messages. This can be consumed in real time across all the User's endpoints to ensure a correct, synchronised representation on all clients for the User.

    Using this consumed status it is also used to represent other Channel Members Read Status and Consumption Point for Messages within a Channel.

    Table of contents:

    • Consumption Horizon
    • Message Read Status

    Consumption Horizon

    A User's Consumption Horizon within a Channel is based on a lastConsumedMessageIndex for a User within a Channel. This references the index property of a Message. A method in the each client SDK is provided allowing the User to send a Consumption Report supplying the User's last consumed Message index for the Channel.

    Note: The Consumption Horizon is not automatically set by Chat: if this is not set, no Consumption Horizon will exist for the User within the Channel. This means that the User's Consumption Horizon (read status) across their clients will not be correctly synchronised.

    To send a Consumption Report, the following code can be used:

    Sending a Consumption Report
    • JavaScript
    //determine the newest message index
var newestMessageIndex = activeChannel.messages.length ? 
    activeChannel.messages[activeChannel.messages.length-1].index : 0;
//check if we we need to set the consumption horizon
if (activeChannel.lastConsumedMessageIndex !== newestMessageIndex) {
    activeChannel.updateLastConsumedMessageIndex(newestMessageIndex);
}
    Sending a Consumption Report

    Note: Consumption report submissions from client endpoints are batched, and not sent per report submission API call. The batch sends are timer based - and by default are submitted every 10 seconds. This send interval time, in seconds, can be configured per Service instance via the REST API using the ConsumptionReportInterval property.

    Note: It is possible to disable the Message Consumption Horizon feature entirely for a Service instance. This is done by configuraing the Service instance resource using the REST API and setting the ReadStatusEnabled property to false. Setting this property to true will enable the feature.

    Message Read Status

    Showing where other Channel Members have read to within a Channel's Messages is achieved by referencing other Members' Channel Consumption Horizon values. This allows for Channel implementations which show who has seen what Messages within the Channel.

    Determing where a Member has consumed up to is done by referencing the lastConsumedMessageIndex of the Member. It is also possible to show when a Member last set their Consumption Horizon by referencing the lastConsumptionTimestamp property of the Member. These properties are available to read statically from the Member instances, but can also be used to show changes to Members' Consumption Horizons in real time by listening for the update event of the Member instances.

    Note: The Consumption Horizon is not automatically set by Chat: if this is not set for a User's Channel Messages, other Channel Members will not be able to know where that Member has read up to within a Channel.

    Here is an example of listening for this event to update the read status for Channel Members:

    Updating Member Read Status in Realtime
    • JavaScript
    //retrieve the list of members for the active channel
var members = activeChannel.getMembers();
//for each member, set up a listener for when the member is updated
members.then(function(currentMembers) {
  currentMembers.forEach(function(member) {
     //handle the read status information for this member
     //note this method would use the provided information 
     //to render this to the user in some way
     updateMemberMessageReadStatus(member.identity, 
                                   member.lastConsumedMessageIndex, 
                                   member.lastConsumptionTimestamp);
  });
});
    Updating Member Read Status in Realtime

    Here is how we would register to listen for Member update events, and then update the relevant Member's message read status:

    //assumes you have a variable names activeChannel for the currently active channel in the UI
activeChannel.on('memberUpdated', function(member) {
   //note this method would use the provided information
   //to render this to the user in some way        
   updateMemberMessageReadStatus(member.identity, 
                  member.lastConsumedMessageIndex, 
                  member.lastConsumptionTimestamp);
});
