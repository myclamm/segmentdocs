Secure Communications
in the Cloud

The latest security practices and full transparency of policies and operations are at the core of the Twilio platform.

Layered SecurityTransparent Security OperationsData Security Controls

Layered Security

  • Physical
  • Network
  • Application
  • Multi-tenancy

Physical

A secure physical facility with round-the-clock surveillance, multi-factor authentication, redundancy zones, and security logging are included not only in the Twilio infrastructure but Amazon Web Services (AWS), where the platform is hosted. AWS complies with AICPA SysTrust, ISO 27001, and other leading physical security frameworks.

Learn more about physical security

Network

Twilio employs best practices for network security. Protecting customer data from application to the platform to thousands of carrier connections around the world. Preventative measures include network firewalls, denial-of-service (DoS), distributed DoS prevention, and network posture assessment.

Learn more about network security

Application

HTTPS-encrypted communication, strong TwiML URL password protection, role-based authorization, and validation of all outbound requests to applications keep Twilio-powered apps protected and available.

Learn more about application security

Multi-tenancy

Only Twilio offers multi-tenancy in tiers. Underlying cloud infrastructure, voice and messaging platform, and Twilio-powered applications are isolated and secure when present on the same server instance. Each customer's activity and data is separate and protected using subaccounts.

Learn more about multi-tenancy security

Transparent security operations

  • Security audits
  • 24/7 incident response
  • Privacy policies

Security Audits

Twilio regularly scans our systems for security vulnerabilities and performs third-party penetration tests several times a year. All access to Twilio production clusters is restricted to Twilio engineers, regularly logged, and audited.

Learn more about audits

24/7 incident response

We strictly follow an incident policy for responding to and reporting of different vulnerability risks. A Security Incident Response Team monitors alerts from upstream vendors, on call twenty-four hours a day, seven days a week.

Learn more about incident response

Privacy policies

Strict data privacy policies block access to personal data and ensure it is only used to deliver the services configured. All Twilio employees are also trained on privacy policies and notified of ongoing updates.

Learn more about data privacy

Data security controls

  • Keep data private
  • PCI and HIPAA ready
  • Safe Harbor compliant

Keep data private

Gain direct control over what voice and messaging data is kept with the Delete API. Manage notifications, calls, recordings, transcriptions, messages, message bodies, and message media and keep data secure.

Read about security control

PCI and HIPAA ready

Published guidelines and recommended techniques to help build applications in compliance with PCI standards and HIPAA Privacy and Security Rules.

Learn more about readiness

Safe Harbor compliant

Twilio adheres to the Safe Harbor data privacy framework covering the collection, use, and retention of personal information received in EU countries and Switzerland. We follow the Safe Harbor principles of notice, choice, onward transfer, security, data integrity, access, and enforcement.

Read about compliance
